# **Домашнее задание №3 (Контроль версий. Семинар 3)**

Git - программа, которая берет на себя вопросы контроля версий проекта. Она сохраняет в памяти не файлы целиком, а только разницу между ними.

 **1. Регистрация пользователя:**

* git config --global user.name _user_

* git config --global user.email _email_

 **2. Основные команды:**

* git init - включение репозитория в данной папке

* git add file_name - начало отслеживания версий

* git status - проверка статуса 

* git commit -m “comment” - применение изменений и оставление комментария 

* git log - посомтреть историю комитов

* git checkout <4-6 символов билда> переход к определенной версии

* git diff - показывает разницу 

* git branch - показывает ветки

* git branch text_formatting - создание ветки text_formatting

* git checkout text_formatting - переход на ветку

* git merge - слияние 2х веток

* git branch -d text_formatting - удаление не нужной ветки

 **3. Символы для разметки:**

* "*" - курсив или (_)

* "**" - полужирный или (__)

* "***" - полужирное курсивное

* "~~" - зачеркнутый текст

Альтернативные символы нужны чтобы совмещать разное форматирование.

* "*_пробел" - список

* "1._пробел" - Нумерованный список

* "#_" - заголовок

**4. Горячие клавиши:**

* "q" - позволяет поднять на уровень выше 

**5. Лекция 3. Работа с удаленными репозиториями:**

* git clone <link_from_GH> - копирование кода с удаленного репозитория
* cd <folder_name> - изменение папки

Отправка на удаленный репозиторий:

1. git remote add origin <link_to_rep> - привязка к удаленному репозиторию
2. git branch -M main - главная ветка
3. git push -u origin main - отправить файлы на удаленный репозиторий

Загрузка с онлайн репозитория себе актуальной версии:
* git pull - стянуть актуальный билд (+merge веток).

* кнопка Forke - сделать на GH полную копию репозитория.

Предложение изменений в чужом проекте:

1. делаем ФОРК интересующего репозитория себе
2. делаем git clone для нашей версии репозиторя
3. создаем ветку с изменениями
4. производим правки, отправляем пуш на свой аккаунт (в форк)
5. В окне на гитхабе появится кнопка отправить Pull request

**6. Чтобы вставить избражение нужно:**

!_[комментарий]_(./путь к картинке) нужно писать без "_"
