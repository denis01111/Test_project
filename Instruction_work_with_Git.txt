Инструкция по работе с Git (команды):

Инициализация локального репозитория:
Выбрать нужную папку и прописать команду 
- git init

Добавить файл к отслеживаю в git
- git add .

Добавить commit (запомнить текущее состояние версии программы)
- git commit -m "Название коммита"

Посмотреть отличия Не сохраненного файла от прошлого файла
- git diff 

Посмотреть журнал всех коммитов
- git log 

Переключаться между коммитами можно с помощью команды 
- git checkout da49 (последние 4 цифры коммита)

Посмотреть на какой ветке находимся 
- git branch

Создать новую ветку 
- git branch name_branch

Переключиться между ветками 
- git checkout name_branch

Обьединить ветки
- git merge name_branch

Удалить ветку, когда все изменения переданы в другую ветку
- git branch -d name_branch

Просмотреть информацию о том какие ветки были объединены
- git log --graph

Переход в другую дирректорию с помощью терминала
- cd c:\\

Чтобы сколнировать файлы с гитхаба к себе на локальный компьютер нужно использовать команду 
- git clone http:// ... ссылка берется с сайта гитхаба

Чтобы создать репозиторий на гитхабе, или залить локальную версию на гитхаб, нужно авторизироваться на сайте и следовать интсрукциям сайта (на каждый случай там расписаны команды)
- git remote add origin http://
- git branch -M main
- git puch -u origin main  - толкнуть локальные изменения(коммиты) на гитхаб
- (выше указан пример, как залить свои изменения на гитхаб)

Чтобы залить изменения с гитхаба на локальную версию нужно ипользовать 
- git pull (но эта команда попытается смержить локальную ветку и гитхабовскую)

Чтобы добавить в свой аккаунт гитхаба чужой репозиторий и производить с ним каку-нибудь работу, то нужно нажать кнопку FORK на сайте В отличии от "git clone" FORK
при его использовании позволяет предложить первоисточнику внести Ваши изменения в свой проект с помощью Pull Requesta
